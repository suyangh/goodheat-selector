# GOODHEAT 暖通产品智能选型工具 - 需求文档

**版本：** v1.0
**日期：** 2026-03-18

---

## 一、项目概述

一款 Web 版暖通产品智能选型工具，用户输入房屋信息、生活习惯及气候条件，系统自动计算并推荐唯一的 GOODHEAT 产品型号。

**目标用户：** 少数经销商/内部人员
**设计原则：** 快速开发、简洁可用

---

## 二、一阶段 MVP（快速版本）

### 2.1 目标
用最少代码实现核心选型功能，1-2 天内可完成。

### 2.2 功能范围

| 模块 | 功能 | 实现方式 |
|------|------|----------|
| 房屋信息 | 建筑类型、面积、年代、末端形式、保温改良 | 单页表单 |
| 生活热水 | 人数、是否有浴缸 | 单页表单 |
| 气候校准 | 期望室温、当地气温（手动选择） | 单页表单 |
| 计算引擎 | 热负荷计算 + 产品匹配 | 内联 JS |
| 结果展示 | 推荐型号 + 关键参数 | 弹窗/模态框 |
| 多语言 | 中文、英文 | JS 对象切换 |

### 2.3 技术栈

```
- HTML5（单文件）
- Tailwind CSS CDN（样式）
- 原生 JavaScript（无框架）
- 数据内联在 HTML 中
```

### 2.4 文件结构

```
goodheat-selector/
└── index.html    # 所有代码 + 数据 + 样式
```

### 2.5 简化点

- ❌ 无 IP 自动定位（手动选择城市）
- ❌ 无经销商地图（仅文字展示）
- ❌ 无 PDF 下载
- ❌ 无多语言框架（仅中英切换）
- ❌ 无后端（纯静态）

## 四、核心算法（两阶段通用）

### 4.1 热负荷计算公式

```
Q_total = (A_floor × q_base) × f_type × f_age × f_retrofit × f_temp
```

| 参数 | 说明 | 取值逻辑 |
|------|------|----------|
| A_floor | 建筑面积 | 用户输入 |
| q_base | 基础热指标 | 散热器 100W/㎡ / 地暖 80W/㎡ / 风盘 90W/㎡ |
| f_type | 建筑类型系数 | 独栋 1.1 / 联排 1.0 / 公寓 0.85 |
| f_age | 年代系数 | <1940:1.4 / 1945-1990:1.0 / 1990-2010:0.8 / >2010:0.6 |
| f_retrofit | 保温改良系数 | 未翻新 1.2 / 换窗 0.9 / 加保温层 0.8（乘积） |
| f_temp | 温度系数 | (T_set - T_design) / 31 |

### 4.2 热水负荷计算

```
无浴缸：V = 人数 × 50L
有浴缸：V = 300L
```

### 4.3 产品匹配逻辑

根据 `Q_total` (kW) 和 `V` (L) 以及末端形式，映射到产品数据库。
